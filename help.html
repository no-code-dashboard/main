No-code-dashboard Help

What is No-code-dashboard App?

No-code-Dashboard App (NCD App) is an easily configurable app that lets you speedily implement of a professional quality dashboard for your projects without any fuss.

The App is based on years of experience of successfully building dashboards to support many different projects.

The first dashboard is built in seconds by just loading data. The dashboard can then be easily configured. The App does not any require specialist skills, equipment or software other than an internet connection and the App. It allows the data held in the collaborative tools (such as Jira, DevOps, Service Now, HP/QC, and good old excel) to be the single source of truth. All stakeholders from execs to PMs to developers/testers use same data to make decisions.

How to use the App?

The main menu on the top of the screen shows the options available.

In non demo mode the options are:

  • TOC shows a table-of-content
  • Load Data allows you to load fresh data. When data is loaded first time a config created automatically. You can modify these configs as described in the following sections. The input data is expected as a csv (comma separated value) file. The App expects the date fields to be in DD-MMM-YYYY or YYYY-MM-DD format. This is to avoid misinterpreting American and European date formats.
  • Modify Layout lets you update the chart title of the data as of date. This is the report date which is used in other charts (W4, Trend and Trend OC),
  • Download Main Config allows you to download the current config. You can use the file later. For some reason the download is not working consistently in all browsers. It download is unsuccessful the config is copied to clipboard and the workaround is to open a new text file, paste the config and save the file
  • Upload Main Config allows you to load (a previously saved) config. In this way you save time when re-configuring charts when data is refreshed.
  • Print PDF creates a PDF version of the dashboard.
  • Help shows you this page
  • Feedback lets you to send an email with your comments.

In demo mode a list of predefined options are shown between TOC and Print PDF

What is a Chart menu?

On the top-right corner of every chart there is a hamburger menu. When you click on this the App will offer following options:

  • Filter Chart shows all the values in the x-axis where you can select the values that you want filtered: check to show and uncheck to hide. Note this option is only shown for the charts that are filterable.
  • Config Chart gives you the option to change the chart as you see fit.This is described in detail in a separate section.
  • Remove Chart deletes the chart. The App asks for a confirmation from you before deleting chart
  • Clone Chart lets you add a copy of the chart at the current position. Once copied you can change the config.
  • Close closes the dialogue.
How to configure charts?

When new data is loaded the App creates a config automatically based on inferred data types. These can be changed using Config chart.

There are some common values for all charts; these are as follows:

  • Chart title lets you change the title of a chart.",
  • Auto title indicates if the title should be automatically set by the App. It is recommended that you keep this checked while you are configuring the charts. Once you are done, you can uncheck this option and enter your value.",
  • Chart size can be small, medium or large. On larger screens, such as PC or laptop, the three types will show appropriately. On a smaller screen, such as mobile, all charts will show in one size.",
  • Chart position lets you re-position a chart. Select a new position and the App move that particular chart to the desired position.

The other values are dependent on the chart type as described in the next section.

How to configure charts using chart types?

The configuration items that are dependent on the chart type are as follows:

  • 2x2 shows the cross tabulated data for two columns. You can select the x and y axes. There is an option of average/sum and count if as described in the next section.
    • ,
  • Data Description describes that input data, i.e. type of data, counts, man and min values.
  • Data Table lists the rows of the input data. You can select the number of rows to display.
  • Date is used when the x-axis has date values. Choose the data table chart type date means the x-axis state at the column ID. You can select the column to show in the x-axis. Different formats available such as DD (day of the month), DDD (day of the week), MMM (month), MMM-YY (specific month) and Wx (weeks from report date). There is an option of average/sum and count if as described in the next section.
  • List Count is useful when a column contains a list of values, for example list of tags or list of linked defects to a test script. You have to indicate a separator for the list, such as comma. The separator defaults to a space if not provided. There is an option count if as described in the next section.
  • List Members is useful when a column contains a list of values as described above. The chart shows the count of tags in the input. You can further define count if value as described in the next section.
  • Note allows to specify a message in the dashboard. It is not a chart but useful in embedding key messages in the dashboard. The demos make use of these to describe the charts.
  • Number is used when the x-axis has numbers. There is an option for youth enter bin to show chart by bin values. The bin if entered must be a comma separated list of numbers in ascending order. You can further define count if as described in the next section.
  • Plan shows the data in a Gantt chart The inputs required are column that has the description to show in the plan, cowman for start date and column for the end date. Columns for start and end dates must contain dates. You can further define count if as described in the next section.
  • Risk shows the data in standard 5x5 risk matrix. You can select the likelihood and impact columns. The App will expect the values 1 to 5 in these columns. You can define the likelihood and impact values as well. There are options of average/sum and count if as described in the next section.
  • State Change is it a chart that shows data on transitions. It expects that the data source is sorted in acceding order for each item. You have to select the columns from where the App picks the values: Time stamp shows the time of the transition while From and To give the values at the point of transition. There is an option to count the number of transition or calculate the total time of transition or average time for transition. There is also an option of count if as described in the next section.
  • String is a is used when the x-axis is neither Date or Number. You can define an order that will make sure (a) the x-axis will be ordered accordingly (b) the item will be shown on the chart even if the value is zero. For instance, you want to show P1 even if zero then enter P1 in order. There are options of average/sum and count if as described in the next section.
  • Trend shows trend of values; for example, test execution trend. You have to specify a date column. You can further define count if, plan and forecast values as described in the next section.
  • Trend OC is very similar to Trend except it calculated the trend using open and close dates. You can further define count if, plan and forecast values as described in the next section.
How to set average/sum, chart filter, plan and forecast?

Following parameters are set using string as an input. To aid your input, a template is displayed when the input field is first clicked. The templates are explained below (where _x_ is shown in the template, a value must be provided).

  • Average/sum where the App uses another column, called column over, that contains numeric data. The App displays the sum or average of the values form this second column.
  • Chart filter has the template action: exclude|include, column-name: column, op: eq|neq, operand: value|[array of values] The template means either exclude or include records where the column-name field contains any item in the array.
  • Plan has the template start: date, end: date, scope-from number, scope-to: max|number, points: line|sigmoid| [number array] The app draws that from start to end going form scope-from to scope to using the points. The point _arraynum_ a numeric ascending array with max value of 1. Examples: [0, 1] draws a straight line. [0, .2, 1] draws the plan using three values, scope-from at the first point, scope-to as the last point and .2 times (scope-to minus scope-from) in the middle.
  • Forecast has two templates. For Chart Type as Trend the template is look-back: number, forecast-to: max|date|number The look-back indicates the number of days previous to the report date used to calculate forecast, forecast-to indicates the end point of the forecast; max means date till the maximum value is reached, date indicates the end date for the forecast, number means a date that is .
  • Forecast for Trend OC has the template is look-back: number, forecast-to: max|date|number The meaning of most parameters are as above except open and close that indicate how the forecast is modified.",
